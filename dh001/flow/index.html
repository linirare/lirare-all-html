代号001 · 对战流程

序号阶段做什么怎么验证备注
一、准备阶段
1装备功能牌 玩家在功能牌仓库选2张牌上阵 检查装备数量=2，每种库存≥1 数量不足阻断，引导去商店/领福利
2开战 玩家点"开始对战" 验证装备2张+库存≥1+不在其他对局中 不通过→弹引导；通过→进匹配
二、匹配阶段
3进入队列 玩家加入匹配池 — 显示"匹配中"+倒计时
4分级匹配 0-5s同段±1小段 → 5-10s放宽±1大段 → 10-15s全段位 — 玩家可随时取消，取消不扣库存
5匹配结果 匹配到真人：创建对局
15s无人：转人机兜底（对玩家无感，不需要选择模式）		 — 对手掉线→也转人机兜底
三、对局创建
6发牌 洗52张牌 → 先发公共牌再发用户 → 每人7张手牌 + 3张公共牌
第1回合：段位高先发；2-5回合：上回输方先发		 — 牌堆不够时接着从弃牌堆抽牌
7暂扣库存 双方装备的2张功能牌库存各-1（标记为"占用"） 库存≥1（已在步骤2验证过） 最终胜退败扣，不是现在扣
8初始化 round=1, hp=3, win=0, 每回合技能次数=0 — 展示对手信息+手牌+公共牌
四、回合循环（每回合重复，共1-5回合）
9补牌 每人手牌补至7张 — 第一回合已发7张，不补
10重置 本回合技能使用次数清零；处理上回合延时效果（如同花信徒/拖延） — start 25s倒计时
11选牌 玩家从手牌中选出N张（第1-3回N=3，第4回N=4，第5回N=5） 选满N张→确认按钮亮；不满N张不可确认 超时→AI代打选牌（仅出牌，不用技能）
12用技能 可选使用功能牌（最多2张/回合，每张牌整场只能用1次） 本回合次数<2 且 该牌未使用 且 回合在可用范围内 且 条件满足 可用时机：回合开始前或选牌阶段；部分牌需要点选目标
13确认出牌 选满N张→点确认 牌来自手牌，数量=N —
14牌型判定 出的N张+3公共牌→取最优5张→算牌型→应用技能加成 — 同比→逐位比较→全相同看花色♠>♥>♦>♣
15本回合结果 胜/负；败方hp-1 — 极罕见同分→单局打平不扣血
16检查终局 hp==0 或 win_count==3 → 比赛结束进结算
否则 → 回到步骤9继续		 — —
五、结算阶段
17处理库存 胜方：退还暂扣的2张 → stock+2
败方：扣除暂扣的2张 → stock保持-2		 — 掉线/投降视同败方
18算奖励 金币×段位倍率 + 钻石固定 + 宝箱进度（仅人机局）+ 角色加成（仅PvE） — 服务端计算，客户端只展示
19算天梯 胜+1星/25分，败-1星/15分，青铜不掉星，连胜/保护判定 — 人机兜底局计入天梯（与PvP规则相同）
20算任务/成就/红点 更新任务进度、牌型统计、红点状态 — —
21生成结算单 生成 settlement_id → 推送结算结果给双方 → 结算失败异步重试 — 5分钟重试3次→邮件补偿
22回主页 关闭结算页 → 刷新主页（资源/段位/红点） — —
六、掉线处理
230-15s 等待重连，暂不代打 — 显示"网络连接中…"
2415-30s AI代打（仅选牌出牌，不用技能） — 重连后恢复控制
2530s+ 判离场→按失败结算（扣除暂扣库存） — 不可重连

整局一副52张牌。牌堆不够时接着从弃牌堆抽牌。第1回合段位高先发，2-5回合上回输方先发。5局3胜，先赢3局结束。
