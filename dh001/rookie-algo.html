代号001 · 新手体验优化算法

核心原则：让新手在前 30 场感受到"我手气不错 / 我能赢 / 这游戏好玩"。算法对玩家不可见——他只觉得自己运气好。对 AI 对手生效（新手期和青铜主要打 AI）。

一、干预维度

干预点做什么玩家感知
手牌品质发 7 张手牌时，提高"有对子/有同花趋势/有顺子趋势"的概率"我手牌不错"
公共牌适配发的 3 张公共牌尽量不破坏玩家手牌趋势"怎么我每次都能凑出来"
补牌友善回合间补牌时，提高与当前手牌相关联的牌的出现概率"抽到我要的了"
AI 手牌弱化AI 的手牌质量略低于玩家（仅前 30 场）"对面好像不太强"
AI 出牌放水AI 偶尔不是选最优牌型，选次优（已设计在胜率锚定中，这里只做放大）"这局我竟然赢了"

二、衰减曲线

场次区间干预强度说明
第 1~3 场（新手局）100%全力保证赢。第一印象决定留存。
第 4~10 场70%仍然有明显优势，但偶尔会输——让玩家知道"我也会输"。
第 11~20 场40%优势收窄。赢一半输一半，但赢的时候牌型往往不错。
第 21~30 场15%几乎感觉不到。只做微调。
第 31 场+0%完全公平。玩家已经养成习惯，不需要照顾了。
强度=100% 是什么意思？不是"100%几率赢"。是干预算法的权重拉满。实际胜率还受 AI 难度/牌运影响。100% 干预下目标胜率约 75%（不是 100%）。

三、手牌品质干预算法

3.1 手牌评分函数

// 7 张初始手牌质量评分（0-100） function handScore(hand[7]): score = 0 // 有对子 +20，有多对子 +30 pairs = countPairs(hand) if pairs >= 1: score += 20 if pairs >= 2: score += 10 // 同花色 ≥4 张 +25（同花趋势） maxSuit = maxSuitCount(hand) if maxSuit >= 4: score += 25 // 有潜在顺子（最大-最小 ≤5 且无重复点数） +15 ranks = sort(uniqueRanks(hand)) if ranks[last] - ranks[first] <= 5 and len(ranks) >= 5: score += 15 // 含 A/K/Q +5 每张 for card in hand: if card.rank >= 12: score += 5 return score

3.2 干预发牌逻辑

// 给新手发牌时调用 function dealWithCare(player, deck): intensity = getFadeCurve(player.matchCount) // 1.0 / 0.7 / 0.4 / 0.15 / 0.0 if intensity == 0: return dealRandom(deck, 7) // 30 场后完全随机 // 不是每次都干预。强度=门槛 if random() > intensity: return dealRandom(deck, 7) // 干预模式：发 3 批牌，选最好的那批 best = null for 3 attempts: candidate = dealRandom(deck, 7) if best == null or handScore(candidate) > handScore(best): best = candidate return best // 注意：3 批牌的牌需要放回牌堆（只是模拟，不是实发）
"发三批选最好"不是真正从牌堆里拿三批——是模拟 3 次，选评分最高的那批作为实际手牌。其余两批的牌放回牌堆重新洗。对牌堆的剩余牌分布没有影响。

四、公共牌适配算法

function dealCommunityWithCare(playerHand, deck, intensity): if random() > intensity: return dealRandom(deck, 3) // 计算玩家手牌的趋势 trend = analyze(playerHand) // trend = { dominantSuit: ♥, hasPair: true, pairRank: K, nearStraight: false } // 发 3 批公共牌，选最"帮到玩家"的那批 best = null bestMatch = -1 for 3 attempts: candidate = dealRandom(deck, 3) match = communityMatchScore(candidate, trend) if match > bestMatch: best = candidate; bestMatch = match return best

4.1 社区牌适配评分

// 公共牌对玩家的"帮助程度" function communityMatchScore(community, trend): score = 0 // 公共牌和手牌有同花色 → +10 每张 score += cardsInSuit(community, trend.dominantSuit) * 10 // 公共牌中有玩家对子的同点数 → +30（凑三条） if trend.hasPair and hasRank(community, trend.pairRank): score += 30 // 公共牌和玩家手牌能凑成顺子趋势 → +25 combined = toRanks(trend.hand + community) if canFormStraight(combined): score += 25 // 公共牌高牌多 → +5 每张（A/K/Q 让玩家"以为"自己很强） for card in community: if card.rank >= 12: score += 5 return score

五、补牌友善算法

function drawCardsWithCare(playerHand, deck, count, intensity): if random() > intensity: return drawRandom(deck, count) // 从牌堆顶看 N*3 张，选对玩家最有利的 N 张 lookahead = min(len(deck), count * 3) candidates = peekTop(deck, lookahead) // 按"与当前手牌的契合度"排序 candidates.sort(byMatchWith(playerHand), desc) // 拿前 N 张 result = candidates[0:count] removeFromDeck(deck, result) return result // 契合度 = 与手牌同花色 + 与手牌能成对 + 与手牌能成顺 function byMatchWith(hand): return function(card): score = 0 if card.suit == dominantSuit(hand): score += 5 if handHasRank(hand, card.rank): score += 3 if nearStraight(hand, card): score += 2 return score

六、AI 手牌弱化算法

// 仅在第 1-10 场生效（新手局 + 青铜前期） function dealAIHand(intensity): // 新手局 100% 弱化，第 10 场以后完全公平 aiIntensity = intensity * clamp(1 - matchCount / 10, 0, 1) if aiIntensity == 0: return dealRandom(deck, 7) // AI 手牌 = 随机发 3 批，选最差的给 AI worst = null for 3 attempts: candidate = dealRandom(deck, 7) if worst == null or handScore(candidate) < handScore(worst): worst = candidate return worst
⚠ AI 弱化仅在"人机兜底"对局中生效。PvP 对局不碰对手手牌。新手局本身只排 AI，所以不存在 PvP 场景。

七、不可见性保障

保障措施怎么防止玩家察觉
不是"100% 发好牌"干预强度控制的是概率，不是保证。强度 100% 时仍有约 25% 场次手牌一般。玩家会认为那是"正常运气波动"。
对比组不存在新玩家不知道"其他人手牌怎样"。他没有对照——你给了他 75% 的好牌率，他以为这是正常游戏。
消退自然30 场逐渐消退。玩家会归因于"我段位高了，对手也强了"而不是"系统不再照顾我了"。
对手是 AIAI 不会抱怨。PvP 对局中不生效。玩家在 PvP 中输了，会认为是"对面太强"而不是"我之前被照顾了"。
不写日志不发"好牌"事件到客户端。客户端收到的就是普通手牌——加密，不懂算法的人反编译也看不出异常。

八、全局开关

// 服务端配置 newPlayerBoost: enabled: true // 总开关：紧急时可一键关闭 maxMatchCount: 30 // 第 31 场起完全关闭 fadeCurve: [1.0, 0.7, 0.4, 0.15, 0.0] // 分段强度 fadeThresholds: [0, 4, 11, 21, 31] // 场次分割点 maxAttempts: 3 // 每次干预模拟几批牌（越大越消耗 CPU，3 足够） targetWinRate: 0.75 // 目标胜率监测线（超过 80% 自动降低强度） // 安全阀：如果玩家胜率超过 85% 持续 5 场，自动降一档强度 safetyValve: winRateThreshold: 0.85 consecutiveMatches: 5 action: "reduceIntensity"
⚠ 安全阀是防作弊的最后防线。如果因为某种原因（bug/异常牌堆）导致玩家胜率远超预期，系统自动拉闸。正常运作不会触发。

实施建议：先上线 AB 测试——10% 新用户走无干预路径作为对照组，90% 走有干预。两周后对比留存率。如果提升不明显→关闭开关；如果提升显著→全量。这是零风险的上线方式。
