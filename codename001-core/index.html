代号 001 · 主页策划案

v0.1 | 2026-07-06 | 功能策划 / UI 交互
本文档以正式项目主页为准；配套线框图说明页面层级和交互入口，不代表最终美术效果。

一文档定位

明确《代号 001》正式主页的定位、信息层级、核心模块、基础交互和首发范围。本文新增主页线框示意图，图中标注与策划规则建立对应关系。

项目内容
适用范围正式项目主页 / 对战大厅界面设计
不包含具体开发实现、资源清单、服务器架构
粒度中等粒度：够落地，不展开到实现细节

二核心目标与设计原则

《代号 001》是轻量化休闲竞技卡牌对战。主页需要让玩家一眼看清身份，一键开始对战，并在开战前完成功能牌配置。

🎯 单核心按钮
只设一个 CTA"开始对战"，点击直接匹配，不给模式选择页。
⚙️ 强制前置配置
功能牌必须配好才能开战，未配置时阻断开战并引导去功能牌页。
📊 信息分层
核心信息在上半屏，运营入口走底栏、弹窗或后续版本。
☀️ 轻量明亮
休闲、干净、圆润，不做沉重信息堆叠。
🚀 首发克制
社交/运营入口首发只开放聊天；赛季/通行证/公会/强运营活动全部后续扩展。

三主页布局

竖屏大厅结构：顶部信息栏 → 主视觉/身份区 → 开始对战 → 功能牌配置区 → 聊天入口 → 底部导航。线框图仅表现首屏核心路径。

四核心交互流程

4.1 进入对战

  1. 进主页 → 展示身份/段位/资源/功能牌配置状态。
  2. 点"开始对战" → 系统校验配置。
  3. 配置未通过 → 弹引导，不匹配，不使用默认牌。
  4. 配置通过 → 直接匹配 → 对战 → 结算 → 回主页刷新资源、段位、奖励、宝箱、功能牌库存与红点。

4.2 功能牌配置

  1. 从主页配置区或底部"功能牌"进入仓库。
  2. 按规则配置指定数量（2 张）→ 回主页显示已上阵牌。
  3. 未完成时"开始对战"可点但弹引导，不进匹配。

4.3 聊天、邮件、活动

  1. 聊天首发开放，主页显示最新一条消息，点开展开面板。
  2. 邮件/公告用红点或弹窗接入，不抢主按钮权重。
  3. 运营活动不占首屏；需要强曝光时走商店/邮件/公告/新增活动条。

六红点规则与刷新

红点规则：仅以下场景强提示红点：可领取、未读、新解锁、可升级。不强提示排行变化、运营入口、礼包推荐、功能牌库存不足（库存不足由开战时阻断引导，不做红点）。

覆盖位置：功能牌（新获得、可配置）、任务/奖励（可领取）、聊天（未读）、邮件（未读/附件未领）、角色（可升级/新解锁）、商店（可领取/宝箱满）。

注：任务/奖励入口暂由结算页和弹窗承接，主页模块表中未单独列出。

刷新时机（事件驱动，不设定时轮询）：

  • 首次进入主页 → 全量刷新资源、身份、功能牌、红点状态。
  • 对战结算返回 → 刷新资源、段位、奖励、宝箱、功能牌状态。
  • 商店/功能牌/角色返回 → 刷新资源、上阵配置、对应入口红点。
  • 聊天、邮件、公告、倒计时 → 由服务端推送、前台恢复检测或低频轮询更新，不设全局定时轮询。
修改说明：去掉了原"定时刷新"，改为纯事件驱动+服务端推送。资源/段位/任务等只在关键事件触发时刷新。

代号 001 · 对战系统策划案

v1.0 | 2026-07-07 | 系统策划 / 一致性优化
对战系统是项目的核心玩法入口，负责从"开始对战"到"整场结算"的完整闭环。

一系统定位

  • 入口：玩家只看到"开始对战"，系统自动匹配，超时自动分配人机，玩家无感。
  • 输出：胜负、奖励、任务进度、天梯变化、角色加成展示。
  • 前置条件：必须配置 2 张功能牌且库存足够。

二对战模式

模式对手入口奖励/天梯定位
PvP 天梯匹配真人开始对战自动匹配正式奖励 + 天梯变化核心竞争模式
人机兜底AI系统匹配不到真人时自动分配（对玩家无感）正式奖励 + 可计入基础天梯
教学AI/固定流程新号引导入口无奖励，不计天梯理解规则

📌 系统需记录对手类型（真人/人机），用于后续排行、反刷和运营分析。

三发牌与战斗规则

牌堆：52 张标准扑克牌，开局每人发 7 张手牌 + 3 张公共牌（整场固定）。

每小局：必须出满 N 张，出的 N 张 + 3 张公共牌取最优 5 张比大小。补牌至 7 张后下一局。N=5 时单局最高消耗 5 张，一局共需 ≤28 张，52 张够用。

  • 5 局 3 胜制，双方初始 3 点生命。
  • 赢家使对手 -1 生命；先赢 3 局或 HP 归零结束。
小局需出牌数
第 1-3 局3 张
第 4 局4 张
第 5 局5 张

四匹配流程

点击开始对战 → 校验配置和库存 → 5-10-15s分段匹配 → 无人转人机 → 暂扣库存 → 战后胜退败扣

五功能牌使用规则

规则说明
带入数量每场 2 张。
库存消耗创建时暂扣，结算胜退败扣；取消/教学不消耗
异常返还无有效结算且非主动离场则退还暂扣库存
使用次数每回合最多用 2 张；每张牌每场只能用 1 次。
跨小局重置回合次数，不重置已用功能牌。

六结算与奖励

以服务端结算为唯一权威。客户端只展示结果，不计算段位、奖励、库存。

每场结算输出：胜负、对手类型、是否计入天梯、金币/钻石/宝箱变化、段位/星数变化、连胜及保护标记、任务进度、角色加成、功能牌消耗/退还结果。胜方退还库存，败方扣除。

段位倍率仅作用于对战基础金币，钻石段位倍率固定为 1.0，不影响其他奖励。

结果金币钻石宝箱进度
PvP 胜利200 × 段位倍率50不积累
PvP 失败80 × 段位倍率20不积累
人机兜底胜利200 × 段位倍率50+2
人机兜底失败80 × 段位倍率20+1
教学无无无

段位倍率首发只影响金币，钻石固定，避免高段位抽卡资源滚雪球。

七UI 与异常处理

  • 对局中隐藏底部导航与非必要入口。
  • 手牌选中需有明确浮起/高亮效果。
  • PvP 出牌超时由服务端代打，客户端只做倒计时展示。
  • PvP 掉线：15s AI代打，再等15s可重连不判负，超30s视为离场。
  • 重连后恢复当前对局状态，不重复发奖。
🔗 跨系统：开战前校验功能牌配置（依赖商店）；结算后输出天梯变化、任务进度、角色加成。

代号 001 · 天梯系统策划案

v1.0 | 2026-07-07 | 系统策划 / 一致性优化
天梯系统承接正式对战后的长期成长反馈。首发不做赛季重置，不引入通行证。

一系统定位

  • 入口：底部"排行"入口进入；结算页可跳转。
  • 输出：段位、小段、星数/积分、排行榜、保护与连胜反馈。

二段位结构

大段小段星数/积分升降规则定位
新手局—3 场人机保护局统一开战入口下 3 场人机对战；正常获奖励，不进排行榜缓冲区
青铜Ⅲ/Ⅱ/Ⅰ每小段 3 星胜+1，败不掉星入门成长
白银Ⅲ/Ⅱ/Ⅰ每小段 3 星胜+1，败-1基础竞争
黄金Ⅲ/Ⅱ/Ⅰ每小段 4 星胜+1，败-1；可连胜奖励主体成长
铂金Ⅲ/Ⅱ/Ⅰ每小段 4 星胜+1，败-1；无连胜高压过渡
大师—1000 分起胜+25，负-15；1500 进传奇高段竞争
传奇—1500 分+按积分排行；首发可不降段荣誉段

4.2 升降段与保护

小段顺序：Ⅲ → Ⅱ → Ⅰ → 升大段。示例：青铜Ⅲ 2/3星胜利 → 青铜Ⅱ 0/3星。

  • 满星胜利 → 晋级下一段，星数重置为 0。
  • 白银+ 在 0 星失败 → 降小段；大段最低小段无保护则降大段。
  • 刚升入白银/黄金/铂金 → 获 1 次大段保护，优先消耗。
  • 青铜永久不掉星，不回新手局。

4.3 跨段位过渡

过渡规则
新手局→青铜打满3场人机保护局自动进入青铜Ⅲ 0/3星
铂金Ⅰ满星→大师晋级大师，初始积分1000
大师→铂金大师积分低于900时掉回铂金Ⅰ3星
大师1000→传奇1500积分达到1500晋级传奇
传奇→大师首发不降段，低于1500仍保留传奇标识
条件变化
普通胜利+1 星；大师+ +25 分
普通失败白银+ -1 星；大师+ -15 分
青铜失败不掉星
青铜/白银/黄金 3 连胜额外 +1 星
铂金+ 连胜无额外星数

四天梯计入范围

模式计入说明
PvP是正式竞争，影响段位和排行。
人机兜底局是计入天梯，AI对局积分按80%折算；后台记录对手类型
教学否不计入，不发奖励。

五排行页

归属底部"排行"入口。排序：大段 → 小段 → 星数/积分 → 胜率/总胜场。不建议首发强红点。

后台口径：至少保留综合天梯、PvP 胜率、人机局占比三个维度。首发页面可只展示综合榜，数据口径先埋好。

输出使用方用途
段位/小段/星数主页、排行、结算身份展示和成长反馈
段位倍率对战结算影响基础金币（钻石固定）
保护/连胜标记结算页说明星数变化来源
🔗 跨系统：天梯接收对战结算输出的胜负结果；段位倍率输出给对战结算影响奖励。

代号 001 · AI 人机策划案

v2.2（合并版） | 2026-07-07 | 设计文档
AI 对手设计的核心不是做出「最强对手」，而是做出「合适的对手」。

一设计目标

  • AI 行为像真人，低难度也不犯低级错误。
  • 难度平滑渐变，调节不可感知。

二胜率锚定 （新）

目标胜率：青铜 65-75% → 白银黄金 55-65% → 铂金大师 50-60% → 传奇 45-55%

四因子：①近期表现（2场调档+3局冷却） ②全局胜率校准（≥30场调基线） ③技能品质校正（装备tier×0.15） ④段位

因子②决定基线，因子①做临时偏移，每场重新计算。

三难度分层

档位技能使用率出牌阶段技能池
简单10%15%T2+T3
普通30%40%全部
困难55%70%T0+T1+T2

段位加成：min(0.3, rankIdx×0.045)。技能品质校正：最高装备 tier×0.15 减难度偏移。

总公式：rankIdx × 3 + aiTier - 品质系数 × 10

四决策逻辑

选牌：穷举最优，全难度统一。

客户端：优先级列表+概率触发。
回合开始优先 lock/delay；出牌阶段优先 destroyer/bomb。强池含更多技能。

服务端：三维评分(o/d/u)+局势因子（对手1血防×3、自己1血攻×4 等）。

评分乘 0.7-1.3 随机系数。📌 待验证。

⚠ PvE 计入天梯：AI 决策在服务端统一结算，客户端仅展示。

五教学与边界

固定牌组（玩家 A♥A♦ 至少一对 vs AI 不成牌型）。AI 不用技能，不计天梯，不消耗资源。

场景处理
AI 技能用完/目标不存在正常出牌
PvP 超时代打仅选牌不出技能
已达难度边界维持不跳档
