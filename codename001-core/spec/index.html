代号 001 · 研发执行规格

📄 v1.0 📅 2026-07-07 🏷️ 执行规格 / 边界对齐 ⚙️ 研发用
本文档不重复策划案中的玩法规则，只规定"服务端/客户端边界、状态流转、异常处理、结算字段"——研发开工前必须对齐的执行层规格。
以下所有规则以服务端实现为准，客户端仅展示和提交操作。

一服务端权威边界

模块服务端计算客户端职责
匹配校验库存 → 创建匹配队列 → 分配对手（真人/人机）→ 返回匹配结果展示匹配中界面 → 收到结果后进入对局
功能牌库存创建对局时暂扣 → 结算时胜退败扣 → 异常检测返还展示库存数量 → 提交装备操作
对战出牌校验 → 牌型计算 → 小局胜负 → 整场胜负 → 超时代打展示手牌 → 提交出牌指令 → 显示服务端计算结果
结算统一产出奖励/段位/任务/宝箱/红点摘要展示结算结果，不做任何数值计算
天梯接收结算结果 → 更新段位/星数/积分 → 升降段判定 → 保护/连胜计算展示当前段位，不做段位计算
任务/签到/宝箱/邮件进度追踪 → 条件判定 → 奖励发放 → 领取防重展示进度 → 提交领取请求 → 展示结果
红点摘要结算时产出红点变化 → 事件触发时刷新展示红点状态 → 点击后通知服务端已读
商店/抽卡钻石校验 → 扣钻石 → 随机选牌/角色 → 库存增加 → 保底计算展示卡池 → 提交抽卡请求 → 播放动画

二核心流程

2.1 开战→结算→回主页 完整链路

开始对战 → 服务端校验配置+库存 → 进入匹配队列 → 匹配真人成功 / 超时转人机 → 服务端创建对局：洗牌→发7张手牌→发3张公共牌 → 对战：每小局出N张→补牌至7张→比最优5张（N=3/3/3/4/5） → 服务端统一结算 → 返回结算结果+红点摘要 → 客户端展示结算页 → 关闭结算 → 回主页刷新指定字段

2.2 匹配流程（服务端）

  1. 客户端提交"开始对战"请求。
  2. 服务端校验：功能牌配置（2张） + 库存是否足够（每种≥1）。
  3. 校验不通过 → 返回错误码 + 引导跳转到功能牌/商店页面。
  4. 校验通过 → 加入匹配队列。
  5. 分阶段匹配：0-5s 精确匹配（同段位±1小段）→ 5-10s 放宽（±1大段）→ 10-15s 全段位 → 15s 超时自动转入人机兜底。
  6. 无论真人/人机，创建成功后服务端暂扣带入功能牌库存（每种 -1，标记已占用）。
  7. 若创建失败（服务器异常等），不扣库存，客户端回到主页。

2.3 结算流程（服务端）

  1. 对局结束 → 服务端计算胜负、奖励、段位变化。
  2. 统一产出结算结果（详见第四节字段表）。
  3. 更新天梯段位/星数/积分。
  4. 更新任务进度。
  5. 计算角色加成（仅 PvE/日常）。
  6. 更新宝箱进度（仅 PvE）。
  7. 产出红点摘要（哪些入口新亮了/灭了）。
  8. 返回完整结算数据给客户端。

2.4 回主页刷新

进入主页/结算返回主页时，客户端请求以下字段：

  • 玩家资源（金币、钻石）
  • 当前段位/小段/星数
  • 上阵功能牌配置
  • 功能牌库存
  • 红点摘要
  • 聊天最新一条消息（可选）

不请求：全量功能牌列表、全量商店数据、排行榜全量等。

三关键状态表

3.0 掉线处理统一规则

时间段系统行为玩家提示可重连
0-15s等待重连，暂不代打"网络连接中…"✅
15-30sAI 代打（仅选牌出牌，不用功能牌）"网络连接中…"✅ 重连后恢复控制
30s+判离场/投降，按失败结算"已断开连接"❌

AI 对局与 PvP 对局掉线规则一致，不做区分。

状态说明客户端展示
idle未匹配显示"开始对战"按钮
matching匹配中匹配动画/倒计时
matched_pvp匹配到真人对手信息 → 进入对局
matched_ai超时转入人机直接进入对局（玩家无感）
failed匹配失败提示并回到主页
cancelled玩家取消匹配回到主页，不扣库存

3.2 对局状态

状态说明处理
created对局创建成功扣除功能牌库存，开始第1局
playing进行中双方选牌/出牌/技能
paused玩家掉线服务端等待15秒，AI代打
finished正常结束进入结算流程
aborted异常终止未产生有效结算，视情况返还库存

3.3 结算状态

状态说明
settled结算完成，可正常发奖
no_settle无有效结算，不发奖励不扣天梯
refunded无有效结算且非玩家责任，返还功能牌

3.4 领取状态（任务/签到/宝箱/邮件）

状态处理
未领取可点击领取
已领取不可重复领取，服务端防重
已过期每日任务/签到过期后不可补领

防重原则：所有领取请求以服务端是否已发放为准。客户端重复点击只返回"已领取"状态，不重复发奖。

玩家类型每日对战钻石收入功能牌消耗抽卡频率库存变化
轻度（仅任务） 3 场人机 任务70 + 签到30 + 宝箱50 ≈ 150💎 6 张/天 约 6-7 天一次十连 正（抽>消耗）
中度（任务+额外） 6-8 场混打 约 250💎 12-16 张/天 约 4 天一次十连 平衡
重度（高活跃） 15+ 场 PvP 约 500💎（含段位倍率） 30+ 张/天 约 2 天一次十连 可能负（消耗>抽）
✅ 首发决策：
  • 段位倍率首发只影响金币，钻石段位倍率固定为 1.0（不随段位增长）。
  • 每日每种 T3 功能牌各补充 3 张（通过每日福利发放）。胜方不消耗功能牌，败方消耗。按 50% 胜率估算：重度玩家(30场/天)实际消耗 15 场×2张=30张/天，补充 24张/天(8种T3×3)，净亏 6张/天；中度玩家(12场)消耗 6场×2=12张，补充24张，净盈12张。
  • 若首发后钻石产出超出预期，调整手段：降低宝箱钻石 / 签到分段奖励 / 任务奖励下调。
