代号 001 · 邮件系统策划案

📄 v2.0 📅 2026-07-07 🏷️ 设计文档 ✉️ 邮件
邮件系统解决三个场景：新手欢迎信+初始资源、运营全服公告/定向消息、附件发放。不做复杂收件箱管理。

一邮件类型

类型发送方触发示例
系统自动程序自动注册、后续系统事件欢迎邮件
全服运营管理员后台手动发送活动公告、补偿
个人运营管理员后台按 UID 发送定向奖励、客服回复

三种邮件在数据结构上没有区别——都是一个 mailItem 对象。区别只在发送方式。

1.1 欢迎邮件

标题：🎉 欢迎加入代号 001 正文：欢迎来到代号 001—— 一款策略与运气并重的扑克对决游戏！ 🎁 附上新手礼包，助你开局顺利。 💡 点击主页左上角头像可以修改你的游戏昵称。 快去功能牌仓库装备好你的牌组，开始第一场对战吧！ 附件：18888🪙 + 1888💎

1888💎 可发一次十连 + 几次单抽，帮新手建初始牌池。18888🪙 保证新手期不缺升级资源。开局冗余保证新手头几天不因资源不足弃游。

二数据结构

{ "id": "k3x7v9_42", "from": "system", "title": "🎉 欢迎加入代号 001", "body": "欢迎来到代号 001……", "items": { "coins": 18888, "gems": 1888 }, "read": false, "claimed": false, "created_at": "2026-07-07 10:30:00" }
字段说明
id唯一标识。全服邮件用 base36(时间戳)_用户ID
from发件人标识。目前只有 system
body支持 \n 换行，客户端渲染为 <br>
items附件（可选）。有附件时显示领取按钮
read是否已读
claimed附件是否已领取
created_at发送时间

三玩家流程与 UI

3.1 操作流程

登录/回主页 → 拉取邮件列表 → 有未读→红点提示 → 点击邮件→展开正文 → 标记已读 → 有附件→领取→钻石金币到账

3.2 UI 布局

┌─────────────────────────┐ │ ✉️ 邮件 [关闭] │ ├─────────────────────────┤ │ ┌───────────────────┐ │ │ │ 📬 欢迎加入代号001 │ │ ← 未读（橙色标记） │ │ 2026-07-07 │ │ │ │ ……正文…… │ │ │ │ 💎 +1888 🪙 +18888 │ │ │ │ [📦 领取奖励] │ │ ← 有附件且未领取 │ └───────────────────┘ │ │ ┌───────────────────┐ │ │ │ 📨 活动公告 │ │ ← 已读（灰色标记） │ │ [领取] → [已领取] │ │ │ └───────────────────┘ │ └─────────────────────────┘

红点：主页右下角邮件 FAB 图标显示未读数量。已读不显示红点。已领取不重复显示领取按钮。列表按创建时间降序排列。

四API 与存储

接口说明
GET /api/auth/mail拉取邮件列表
POST /api/auth/mail/read/:id标记已读
POST /api/auth/mail/claim/:id领取附件（幂等，防重）
POST /api/auth/admin/mail全服邮件发送
POST /api/auth/admin/mail/uid/:uid个人邮件发送

每次进入主页/切换页面时拉取，不设定时轮询。领取操作幂等，第二次请求不重复发资源。

4.1 存储限制

当前邮件直接存在用户 mail 数组中，无数量上限。建议兜底：

  • 邮件总数上限 100 条
  • 超出时删除最早已读且已领取的邮件
  • 未读/未领取邮件不被自动删除

（当前未实现此限制。）
