一文档定位

明确《代号 001》正式主页的定位、信息层级、核心模块、基础交互和首发范围。本文新增主页线框示意图，图中标注与策划规则建立对应关系。

项目内容
适用范围正式项目主页 / 对战大厅界面设计
不包含具体开发实现、资源清单、服务器架构
粒度中等粒度：够落地，不展开到实现细节

二核心目标与设计原则

《代号 001》是轻量化休闲竞技卡牌对战。主页需要让玩家一眼看清身份，一键开始对战，并在开战前完成功能牌配置。

🎯 单核心按钮
只设一个 CTA"开始对战"，点击直接匹配，不给模式选择页。
⚙️ 强制前置配置
功能牌必须配好才能开战，未配置时阻断开战并引导去功能牌页。
📊 信息分层
核心信息在上半屏，运营入口走底栏、弹窗或后续版本。
☀️ 轻量明亮
休闲、干净、圆润，不做沉重信息堆叠。
🚀 首发克制
社交/运营入口首发只开放聊天；赛季/通行证/公会/强运营活动全部后续扩展。

三主页布局

竖屏大厅结构，从上到下：顶部信息栏 → 主视觉/身份区 → 开始对战 → 功能牌配置区 → 聊天入口 → 底部导航。线框图仅表现首屏核心路径。

四核心交互流程

4.1 进入对战

  1. 进主页 → 展示身份/段位/资源/功能牌配置状态。
  2. 点"开始对战" → 系统校验配置。
  3. 配置未通过 → 弹引导，不匹配，不使用默认牌。
  4. 配置通过 → 直接匹配 → 对战 → 结算 → 回主页刷新所有状态。

4.2 功能牌配置

  1. 从主页配置区或底部"功能牌"进入仓库。
  2. 按规则配置指定数量 → 回主页显示已上阵牌。
  3. 未完成时"开始对战"可点但弹引导，不进匹配。

4.3 聊天、邮件、活动

  1. 聊天首发开放，主页显示最新一条消息，点开展开面板。
  2. 邮件/公告用红点或弹窗接入，不抢主按钮权重。
  3. 运营活动不占首屏；需要强曝光时走商店/邮件/公告/新增活动条。

六红点规则与刷新

通用逻辑：有未读 / 可领 / 可升级 → 亮红点；查看或操作后消失。覆盖：功能牌、任务/奖励、聊天、邮件、角色、商店/礼包、排行、运营入口。

注：任务/奖励入口暂由结算页和弹窗承接，主页模块表中未单独列出。

刷新时机：

  • 首次进入主页 → 全量刷新资源、身份、功能牌、红点状态。
  • 对战结算返回 → 刷新资源、段位、奖励、宝箱、功能牌状态。
  • 商店/功能牌/角色返回 → 刷新资源、上阵配置、对应入口红点。
  • 定时刷新 → 仅聊天/邮件/倒计时。

七UI 风格

方向要求
整体明亮休闲、欧美卡通、圆润干净
布局节奏上半屏角色&段位，中部开始对战，下半屏功能牌&聊天
按钮主按钮颜色最强，辅助按钮降低饱和度
图标图标配简短文字，不单靠图标
动效主按钮/奖励/升段加强；普通入口克制
适配竖屏优先，底栏适配安全区，信息不贴边
