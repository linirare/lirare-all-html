07Learnings

沉淀出的工作方法

市场数据：先校验时间线

上线时间、排名、官方可验证信息优先；2025 才上线的游戏不能用 2023-2024 数据。

搜索：多源交叉验证

百度 / Tavily / 引力引擎 / DataEye / AppMagic 等按场景组合，不把单一来源当真相。

自动化：前置条件也要自动化

舆情 cron 不能假设 Chrome 已启动；必须用计划任务先开远程调试 Chrome。

产出：报告必须能分享

HTML-PPT 不只是本地文件，要走 GitHub Pages，电脑和手机都能看。

08Pitfalls

踩过的坑：现在已经变成规则

Windows 命令行 JSON

中文和嵌套 JSON 容易被 PowerShell/CMD 转义搞坏，大参数改文件或 API 直调。

Gateway 重启

SIGUSR1 热重启不刷新 modelRegistry；图片模型、输入能力变更必须完全重启。

图片输入 / 生图

模型不支持 image input、MiniMax 额度超限、OSS URL 过期，都要快速切换通道。

网页抓取

Playwright / Cookie 容易触发验证；能用现有登录态 + CDP 就别硬撞反爬。

消息投递

子 Agent 继承 wecom delivery context 会重复发，跨渠道任务要明确 target。

HTML-PPT

自定义 CSS 不能覆盖 `.slide{position:absolute;inset:0}`，否则页面会堆叠溢出。
